## Benchmark Results (vulnerable_app.py)

- Multi-agent: 127 findings, 97.4s, $0.23
- **Multi-agent finds 10.6x more findings**
- **100% coverage overlap** (generalist findings fully contained in multi-agent)
- **115 unique findings** only found by multi-agent
- All 6 categories covered by both modes

## Ronda 4: Negotiation Round (New in orchestrator.py)
- After synthesis, identifies low-consensus findings
- For disputed findings, agents with disagreeing positions debate:
  - State position with evidence
  - Rebut opposing arguments
  - Converge or maintain position
- Transcript recorded in round_data

## Architectural Decisions
- 6 specialized agents (security, architecture, quality, performance, ux, vision)
- 3 debate rounds (analysis → cross-debate → refinement)
- Optional Round 4 for negotiation
- Given-New communication protocol (Inverted Pyramid format)
- Two-layer design: generic multi-agent framework + code review demonstration
- SSE streaming for real-time UI updates (not WebSocket)
- Negotiation only for low-consensus findings after synthesis
