## Root Cause
- OpenAI SDK v2.x validates API key at `AsyncOpenAI()` instantiation
- Tests creating agents without the mock fixture failed because `settings.qwen_api_key` defaults to `""`

## Fixes Applied
1. Added `os.environ.setdefault("OPENAI_API_KEY", "sk-test-fake-key-for-tests")` in `conftest.py` at module level
2. Added module-level `patcher = patch("backend.agents.base_agent.AsyncOpenAI")` in `test_agents.py` to cover tests that create agents directly
3. Updated impact value assertions to Spanish (`Crítico`, `Alto`, `Medio`, `Bajo`) matching `_normalize_impact()` output
4. Fixed `round1` → `round_1` in council test assertion
5. Added `mock_session.commit = AsyncMock()` in API test fixture
6. Fixed indentation bug in `test_parse_single_finding`
